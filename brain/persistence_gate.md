# persistence_gate.md — Серия подряд > hit_ratio

Находка 2026-05-31 (скрипт `scripts/analyze_persistence.py`).

## ПРОБЛЕМА

Текущий ранкинг = `avg_spread × hit_ratio`. Но `hit_ratio` (доля часов выше порога
за 24ч) НЕ различает:
- один сплошной блок 8ч подряд (вход безопасен)
- 8 разбросанных часов по 1ч (вход = убыток)

Break-even требует N часов **подряд**. Если спред живёт 3ч, а break-even 7ч → минус.

## ВЫВОД

- **НЕ снижать SPREAD_ENTRY_THRESHOLD** — впустит больше BCH-подобных убыточных.
- Правильно: **гейт персистентности** — вход только если cumulative spread за
  макс. серию подряд ≥ fee × запас (напр. 1.3×).
- Метрика: walk хронологического ряда signed-спредов, найти max run выше порога,
  суммировать спред в нём, сравнить с комиссией.
