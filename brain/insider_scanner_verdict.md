# insider_scanner_verdict.md — Бэктест InsiderScanner: инсайдеры реальны, follow-стратегия структурно мертва

Триггер: переанализ 2026-06-10. Юзер: «бросил не из-за автоматизации, а потому что не нашёл закономерности — цифры слишком случайные». Бэктест подтвердил его интуицию И нашёл причину.

## ДАННЫЕ
- `/root/projects/InsiderScanner/database/scanner.db`, таблица signals: **1,096,206 сигналов** (2026-03-17 → 2026-04-18).
- Качественные (FRESH ENTRY / WHALE BET / ACCUMULATION): 42,652 сигнала, 1,612 рынков, 2,408 CLOB-токенов.
- Резолюции дотянуты с Gamma API (`?clob_token_ids=...&closed=true` — параметры ПОВТОРОМ, не через запятую; `closed=true` обязателен для закрытых). Покрытие 2,408/2,408.
- Сторона ставки = позиция token_id в clobTokenIds рынка. Слаги в DB самодельные (`4pt7` vs реальный `47`) — джойн только по token_id.

## МЕТОД
Unit = (рынок, сторона), вход по цене первого сигнала, не-спорт. Метрика: win-rate vs implied (= avg price), ROI на $1 flat-stake.

## РЕЗУЛЬТАТЫ (market-dedup, non-sports)
| Срез | units | win | implied | edge | ROI |
|---|---|---|---|---|---|
| ВСЕ качественные | 985 | 53.3% | 54.7% | −1.4% | −1.1% |
| FRESH ENTRY | 855 | 51.8% | 53.9% | −2.1% | −3.3% |
| WHALE BET | 459 | 61.2% | 58.1% | +3.2% (1.4σ) | +23.8% |
| **Лонгшоты <10¢** | **159** | **13.2%** | **5.4%** | **+7.8% (4.3σ)** | **+194%** |
| └ WHALE <10¢ | 32 | 21.9% | 5.6% | +16.3% (4.0σ) | +282% |

Лонгшот-edge устойчив: time-split (март +8.7% / апрель +6.9%), сильнее в 1-5¢ (+9.6%, 4.9σ), победители размазаны по категориям (не один ивент). **Инсайдеры на Polymarket реальны** — их дешёвые ставки выигрывают в 2.4× чаще implied.

## ПОЧЕМУ FOLLOW НЕ РАБОТАЕТ (ключевой замер)
Вход по реальной цене из CLOB prices-history после сигнала:
| Вход | entry px | edge | ROI |
|---|---|---|---|
| Цена инсайдера | 5.4% | +7.8% | +194% |
| **t+0 (первая точка после)** | **14.6%** | **−1.4%** | **−11%** |
| t+2min | 13.6% | −0.4% | +10.8% (шум, t≈0) |
| t+10min | 13.2% | ≈0 | −5.9% |
| фильтр entry<10¢ | 5.0% | +1.8% (0.9σ) | −10.3% |

**Маркет-ордер инсайдера сам и есть репрайсинг**: он съедает тонкий стакан лонгшота, цена x2.7 мгновенно, к моменту детекта (on-chain блок) ловить нечего. Фильтр «вход только если ещё <10¢» отбирает ровно те случаи, где инсайдер НЕ двинул цену = где сигнала нет. Это математически объясняет «цифры случайные»: по достижимым ценам они и есть случайные.

## УТОЧНЕНИЕ (вопрос юзера: «не все же инсайдеры двигают цену»)
Разбор per-unit вместо средних (средние тянули выбросы):
- **57/158 юнитов — аномальные данные**: рыночная цена до сигнала отличается от записанной цены сделки >50% (поле price ≠ market price в момент детекта). На рыночных ценах эти юниты edge не имеют (win 22.8% vs 29.3%).
- **Из консистентных 101: у 91% (92) цена НЕ прыгает** (≤1.2×) — войти по цене инсайдера МОЖНО. Но эти исполнимые сигналы не выигрывают: win 6.5% vs 5.5% implied = edge 0.
- **Прыгнувшие 8 юнитов: win 25%** — прыжок цены сам и есть маркер информированной сделки (срочная инфа → агрессивный удар по стакану). Парадокс follow: исполнимость и информативность взаимоисключающи.
- **Единственный намёк**: исполнимые ставки ≥$700 — win 11.6% vs 5.6%, ROI +57%/юнит, но **1.7σ на 43 юнитах (5 побед) = недоказуемо** после множественных сравнений. Проверка дешёвая: перезапустить сбор сканера на 2-3 мес и пересчитать ИМЕННО этот срез (~40 юнитов/мес).

## ВЫВОДЫ
1. **Insider-following / copy-trading на Polymarket — структурно мёртв** для фоловера при любой скорости (включая 0-латенси): импакт уже в цене. НЕ переоткрывать.
2. Кто зарабатывает на этой структуре: сам инсайдер; **MM, продающий лонгшоты дороже fair** (наша MarketMakerBot-парадигма — но это значит НЕ котировать узко в тонких лонгшотах против информированного потока — adverse selection!); структурные арбитражёры.
3. Практическая ценность находки для MM: **сигнал InsiderScanner = индикатор токсичного потока**. Видим FRESH/WHALE в лонгшоте → снять котировки/расширить спред в этом рынке. Это защитный модуль для MM-бота, не атакующая стратегия.
4. Датасет и пайплайн (fetch по token_id + prices-history) переиспользуемы: /tmp/backtest_insider.py, analyze_insider2.py, slippage_test.py (скопировать в InsiderScanner/research/ при необходимости).
